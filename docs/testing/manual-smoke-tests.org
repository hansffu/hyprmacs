#+title: hyprmacs Manual Smoke Tests
#+date: 2026-04-16

* Purpose
Quick manual checks for each prototype checkpoint in the nested Hyprland environment.

* Prerequisites
- Run nested Hyprland with =just run=.
- Load plugin with =just load=.
- Load Emacs package with =just emacs=.

* Task 1 Bootstrap Smoke
- Confirm plugin load prints a hyprmacs bootstrap notification.
- Confirm Emacs can load =hyprmacs.el= without errors.

* Task 3 Registry Smoke
- Open two =foot= windows and move focus between them.
- Verify plugin logs show event lines and =state-dump= lines.
- Verify Emacs window is present in logs but marked =eligible=false=.

* Task 5 IPC Smoke
- In Emacs run:
  - =M-x hyprmacs-manage-current-workspace=
  - =M-x hyprmacs-request-state=
  - =M-x hyprmacs-dump-state=
- Notes:
  - =hyprmacs-manage-current-workspace= auto-connects if needed.
  - Interactive workspace commands infer current workspace from Hyprland active workspace.
- Verify plugin logs include =ipc recv= and =ipc send= entries.
- Verify =*hyprmacs-state*= updates with live workspace state.

* Task 6 Hidden-Client Spike Smoke
- Ensure at least one eligible client exists (e.g. =foot=).
- In Emacs run:
  - =M-x hyprmacs-debug-hide-client=
  - enter client ID like =0x...=
  - =M-x hyprmacs-debug-show-client=
  - enter same client ID
- Validate for =foot= and one GUI app:
  - Client disappears after hide command.
  - Client process remains alive.
  - Client returns with state intact after show command.
- Additional checks:
  - Switch away and back to workspace while hidden.
  - Close a client while hidden and ensure no plugin crash.

* Task 8 Multi-Client Layout Smoke
- Start at least two managed clients (for example two =foot= windows).
- In Emacs run:
  - =M-x hyprmacs-manage-current-workspace=
  - =M-x hyprmacs-request-state=
  - =M-x hyprmacs-dump-state=
- Display two managed buffers in two Emacs windows:
  - split Emacs window
  - switch each pane to a different =*hyprmacs:...*= buffer
- Trigger a layout publish:
  - automatic: enabled by =hyprmacs-manage-current-workspace=
  - manual fallback: =M-x hyprmacs-sync-layout=
- Validate:
  - both visible managed clients are on workspace 1 and moved to matching rectangles
  - non-visible managed clients move to =special:hyprmacs-hidden=
  - overlapping Emacs panes are rejected with a layout error (no partial mutation)

* Notes
- Task 6 uses temporary debug commands and a special workspace (=special:hyprmacs-hidden=) for hide/restore.
- This is a spike path and may change before layout integration.
